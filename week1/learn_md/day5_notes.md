# Day 5 Notes

## 今天学习的内容
今天我对简化版 RAG 系统做了两个优化：第一是过滤掉 score 为 0 的无关 chunks，第二是优化最终回答格式，让输出更像真正的问答系统。

## 我今天做了什么
1. 修改了检索函数，只保留 score 大于 0 的结果
2. 修改了 `rag_demo.py` 的回答生成逻辑
3. 让程序输出“回答 + 依据 + 总结”的结构化结果
4. 保留参考 chunks，方便查看回答依据

## 我对今天优化的理解
之前程序虽然能回答问题，但是输出比较生硬，而且有时会把完全不相关的 chunk 也带进结果里。今天的优化让回答结果更清晰，也让参考依据更干净。

## 我的实验观察
### 问题：chunk_size 和 chunk_overlap 有什么作用
这个问题在知识库里有明确答案，所以程序可以检索到高分 chunks，并生成较清晰的结构化回答。

### 优化效果
- 回答格式更清晰
- 依据更容易看懂
- 无关的 score = 0 chunks 不再出现在结果中

## 当前程序的局限
1. 检索仍然依赖简单字符片段重合，不是真正语义检索
2. 回答仍然是从句子中选择和拼接，不是真正的大模型生成
3. 有时依据中的句子会出现重复信息或者不完整句子

## 今天的收获
今天我进一步理解了 RAG 系统不仅要能“检索到内容”，还要能把结果组织成更自然、更清晰的回答格式。
